The Science (condensed)

For the full version, read README.md in this folder. These are the load-bearing concepts.

1. OS display scale is invisible to your CSS.

Windows 175% / macOS Retina scaling rescales physical pixels into CSS pixels before your stylesheet ever runs. A 4K monitor at 200% reports the same ~1920 CSS px viewport as a 1080p monitor at 100%. You don't need to detect scale — just build for a range of CSS widths.

2. Browser zoom is the variable that actually changes layout.

Ctrl + multiplies every CSS pixel value. A user at 150% zoom on a 1280px window sees layout as if the viewport were ~853px wide. Honor it: use em in your media queries and rem in your font sizes.

3. clamp(MIN, PREFERRED, MAX) replaces 90% of media queries.

Instead of "size is 16px below 768 and 20px above," write clamp(1rem, 0.9rem + 0.5vw, 1.25rem). The value ramps smoothly between breakpoints. No jumps, no extra rules.

4. A modular scale + fixed spacing scale = visual consistency.

All font sizes derive from one base × ratio. All spacing comes from a 4px-based scale. Use the Type & Space Scales tab to generate yours. After that, never invent a number — pick the token.

5. Layout = 5 primitives, not 50 unique flex/grid rules.

Stack (vertical rhythm), Cluster (horizontal wrap), Sidebar (collapsing column), Switcher (row↔column flip), Auto-grid (responsive cards). Most sites need nothing else. See foundation.css.

6. Container queries beat media queries for components.

A card in a sidebar shouldn't read the viewport — it should read its own container. container-type: inline-size + @container (min-width: 30rem) makes a component truly portable.

7. Why AI-generated CSS looks bad (and how to fix it).

Models pick pixel literals (padding: 20px 40px) because that's what most tutorials show. They duplicate values, mix scales, fear small text. Paste ai-rules.md at the top of every prompt and the issue mostly disappears.

8. Test matrix that catches everything.

320px, 768px, 1280px, 1920px viewport widths × 100% / 150% / 200% browser zoom × light / dark mode. Twelve combinations. If they all look right, the page works on every laptop and every monitor anyone has.
