It is incredibly common for AI to generate "awkward" UI. When left to its own devices, AI tends to mix different layout methods—throwing in random margin-top: 15px here, an absolute position there, and forgetting to use proper Flexbox alignment. This results in things looking slightly off-center and disjointed.

To fix this in React, you need to stop asking the AI to "design a page" and start forcing it to use a Strict Flexbox/Grid Layout System.

2. The Login Page (Perfectly Centered)
The trick to a perfect login page is making the wrapper 100vh (viewport height) and using Flexbox to lock the card dead center.

JavaScript
import React from 'react';

export default function LoginPage() {
  return (
    <div style={{
      display: 'flex',
      alignItems: 'center',       // Centers vertically
      justifyContent: 'center',   // Centers horizontally
      minHeight: '100vh',
      backgroundColor: 'var(--bg)',
      padding: '24px'
    }}>
      <div style={{
        backgroundColor: 'var(--surface)',
        border: '1px solid var(--border)',
        borderRadius: '8px',
        padding: '32px',
        width: '100%',
        maxWidth: '400px',        // Stops it from getting too wide
        display: 'flex',
        flexDirection: 'column',
        gap: '16px',              // Perfectly spaces out the form inputs
        boxShadow: '0 4px 6px -1px rgba(0, 0, 0, 0.05)'
      }}>
        <h2 style={{ textAlign: 'center', marginBottom: '8px' }}>Welcome Back</h2>
        <input placeholder="Email" style={{ padding: '12px', borderRadius: '6px', border: '1px solid var(--border)' }} />
        <input placeholder="Password" type="password" style={{ padding: '12px', borderRadius: '6px', border: '1px solid var(--border)' }} />
        <button style={{
          backgroundColor: 'var(--primary)', color: 'white', padding: '12px', borderRadius: '6px', border: 'none', fontWeight: 'bold', cursor: 'pointer'
        }}>Log In</button>
      </div>
    </div>
  );
}
3. The Dashboard Layout (Sidebar + Content)
Dashboards look awkward when the sidebar and main content aren't linked. We use a master Flex container (flexDirection: 'row') to keep them perfectly side-by-side.

JavaScript
import React from 'react';

export default function DashboardLayout({ children }) {
  return (
    <div style={{ display: 'flex', minHeight: '100vh', backgroundColor: 'var(--bg)' }}>
      
      {/* Sidebar: Fixed width, borders on the right */}
      <aside style={{
        width: '260px',
        backgroundColor: 'var(--surface)',
        borderRight: '1px solid var(--border)',
        padding: '24px',
        display: 'flex',
        flexDirection: 'column',
        gap: '16px'
      }}>
        <h3 style={{ color: 'var(--primary)', fontWeight: 'bold', marginBottom: '24px' }}>My App</h3>
        <nav style={{ display: 'flex', flexDirection: 'column', gap: '8px' }}>
          <button style={{ textAlign: 'left', padding: '8px 12px', background: 'none', border: 'none', cursor: 'pointer', borderRadius: '4px' }}>Dashboard</button>
          <button style={{ textAlign: 'left', padding: '8px 12px', background: 'rgba(37, 99, 235, 0.1)', color: 'var(--primary)', border: 'none', cursor: 'pointer', borderRadius: '4px' }}>QR Builder</button>
        </nav>
      </aside>

      {/* Main Content Area */}
      <main style={{
        flex: 1,                  // Takes up the remaining space
        padding: '48px',
        display: 'flex',
        flexDirection: 'column',
        gap: '24px'               // Spaces out dashboard widgets perfectly
      }}>
        {children}
      </main>
      
    </div>
  );
}
